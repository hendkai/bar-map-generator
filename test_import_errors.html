Configuration Import Error Handling Tests

This page provides test cases for verifying the error handling in the import configuration functionality.

Test Cases:

1. Invalid JSON (malformed)

2. Missing Required Fields

3. Invalid Map Size

4. Invalid Terrain Type

5. Invalid Player Count

6. Value Out of Range (noiseStrength)

7. Value Out of Range (metalSpots)

8. Non-Numeric Value

9. Malformed Base64

10. Empty String

11. Valid Configuration (should succeed)

This is a valid base64-encoded configuration for 1024x1024 continental terrain with 4 players.

How to Test:

  1. Open bar_map_generator.html in a browser
  2. For each test case above, copy the content from the textarea
  3. Paste it into the "Import from Text" textarea in the main application
  4. Click "Import from Text" button
  5. Verify the appropriate error message is displayed
  6. For test case 11, verify it successfully imports

Expected Results:

  • Tests 1-8: Should show specific error messages describing what's wrong
  • Test 9: Should show error about invalid base64 characters
  • Test 10: Should show error about empty configuration
  • Test 11: Should successfully import and show success message
